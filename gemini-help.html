Enable Gemini Nano in Chrome

PromptKeeper uses Gemini Nano, Chrome's built-in local AI model. This feature is experimental and currently requires manual activation.

Requirement: Chrome Version 143+ (Dev/Canary recommended).

Step 1: Enable Chrome Flags

  1. Open chrome://flags ↗
  2. If the link doesn't open, copy and paste it into a new tab.
  3. Search for "Prompt API for Gemini Nano" (#prompt-api-for-gemini-nano) and set it to Enabled.
  4. Search for "Enables optimization guide on device" (#optimization-guide-on-device-model) and set it to Enabled BypassPerfRequirement.
  5. Click the Relaunch button at the bottom of the screen.
Tip: If the AI is still missing after relaunching, proceed to Step 3 below.

Step 2: Download the Model

  1. Open chrome://components ↗
  2. If the link doesn't open, copy and paste it into a new tab.
  3. Find Optimization Guide On Device Model in the list.
  4. Click Check for update.
    • If it says "Component not updated" or "New", wait a few minutes. It is downloading (~1.5GB) in the background.
    • When it says "Up-to-date" (and shows a version > 0.0.0.0), it is ready.

Step 3: Advanced Troubleshooting (Stuck Download?)

If the model doesn't download or "Optimization Guide" is missing:

  1. Open the Developer Console (Ctrl/Cmd + Shift + J).
  2. Run this command: await window.ai.createTextSession();
  3. Return to chrome://components and check again.

Step 3: Verify

Return to PromptKeeper and refresh the page. The status should now say ✅ AI Ready.
